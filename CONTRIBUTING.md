# Contributing to awesome-express

Thank you for your interest in contributing to awesome-express! This document provides guidelines and instructions for contributing.

## Code of Conduct

Please be respectful and considerate of others when contributing to this project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/awesome-express.git`
3. Install dependencies: `npm install`
4. Create a feature branch: `git checkout -b feature/my-feature`

## Development Workflow

1. Make your changes
2. Run tests: `npm test`
3. Lint your code: `npm run lint`
4. Commit your changes: `git commit -am 'Add some feature'`
5. Push to the branch: `git push origin feature/my-feature`
6. Submit a pull request

## Pull Request Process

1. Update the README.md or documentation with details of changes if appropriate
2. Update the ROADMAP.md if you've implemented a planned feature
3. The PR should work with all tests passing
4. Your PR will be reviewed by maintainers who may request changes

## Feature Requests

If you have ideas for new features:

1. Check the ROADMAP.md to see if it's already planned
2. Open an issue with the "feature request" tag
3. Describe the feature and its benefits clearly

## Bug Reports

When reporting bugs:

1. Check if the bug has already been reported
2. Use the bug report template
3. Include clear steps to reproduce
4. Include your environment details (OS, Node version, etc.)

## Coding Standards

- Follow the existing code style
- Write tests for new features
- Add JSDoc comments for public APIs
- Keep commits focused and atomic
